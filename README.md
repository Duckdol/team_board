## 🎯 프로젝트 소개

**팀 소통 게시판**은 회사나 조직에서 팀별로 소통할 수 있는 웹사이트입니다. 

### 🌟 어떤 웹사이트인가요?
- 📝 **팀별 게시판**: 개발팀, 디자인팀, 기획팀 등 팀별로 게시글을 작성하고 소통
- 📱 **모든 기기 지원**: 컴퓨터, 태블릿, 스마트폰에서 모두 사용 가능
- 🎨 **깔끔한 디자인**: GitHub(깃허브)처럼 깔끔하고 사용하기 쉬운 인터페이스
- 🔍 **검색 기능**: 원하는 게시글을 쉽게 찾을 수 있음

---

## 💡 용어 설명

**개발을 모르는 분들을 위한 쉬운 용어 설명입니다!**

### 🔧 개발 도구들
- **GitHub**: 코드를 저장하고 관리하는 온라인 창고 (구글 드라이브 같은 곳)
- **Netlify**: 웹사이트를 인터넷에 올려주는 서비스 (웹호스팅)
- **Next.js**: 웹사이트를 만들기 위한 도구 (워드프레스 같은 것)
- **TypeScript**: 프로그래밍 언어 (웹사이트 만드는 언어)

### 📁 파일 용어들
- **Repository (레포지토리)**: GitHub에서 프로젝트를 저장하는 폴더
- **Build (빌드)**: 우리가 만든 코드를 웹사이트로 변환하는 과정
- **Deploy (배포)**: 만든 웹사이트를 인터넷에 올리는 것

## 📋 단계별 가이드

### 1단계: GitHub에서 코드 가져오기 ⬇️

**목표**: 이미 만들어진 웹사이트 코드를 내 컴퓨터로 가져오기

1. **GitHub 계정 로그인**
   - [github.com](https://github.com)에 접속하여 로그인

2. **새 저장소 만들기**
   - 오른쪽 위 `+` 버튼 클릭 → `New repository` 선택
   - Repository name: `team-board` 입력
   - `Public` 선택 (무료로 사용하려면)
   - `Create repository` 버튼 클릭

3. **코드 업로드하기**
   - 컴퓨터에서 명령 프롬프트(Command Prompt) 또는 터미널 열기
   - 프로젝트 폴더로 이동
   ```bash
   cd team-board
   git remote add origin https://github.com/본인아이디/team-board.git
   git push -u origin main
   ```

> 📸 **중요**: 위의 `본인아이디` 부분을 실제 GitHub 아이디로 바꿔주세요!

---

### 2단계: Netlify에서 웹사이트 배포하기 🌐

**목표**: GitHub에 올린 코드를 실제 웹사이트로 만들어 인터넷에 공개하기

#### 2-1. Netlify 설정

1. **Netlify 로그인**
   - [netlify.com](https://netlify.com)에 접속
   - `Log in` 버튼 클릭
   - `GitHub`으로 로그인 선택 (계정 연동)

2. **새 사이트 추가**
   - `Add new site` 버튼 클릭
   - `Import an existing project` 선택
   - `Deploy with GitHub` 선택

3. **저장소 선택**
   - 권한 승인 후 저장소 목록에서 `team-board` 찾기
   - `team-board` 클릭하여 선택

#### 2-2. 배포 설정 확인

**Netlify가 자동으로 설정을 감지합니다:**

- **Site name**: 원하는 이름으로 변경 가능 (예: `my-team-board`)
- **Branch to deploy**: `main` (기본값)
- **Build command**: `npm run build` (자동 감지)
- **Publish directory**: `.next` (자동 감지)

#### 2-3. 배포 시작

1. **배포 시작**
   - `Deploy site` 버튼 클릭
   - 배포 과정 시작 (3-5분 소요)

2. **배포 상태 확인**
   - `Deploys` 탭에서 진행 상황 확인
   - 녹색 체크 표시가 나오면 성공!

3. **웹사이트 주소 확인**
   - 배포 완료 후 상단에 웹사이트 주소 표시
   - 예: `https://amazing-name-123456.netlify.app`

---

### 3단계: 웹사이트 확인하기 ✅

**목표**: 만든 웹사이트가 잘 작동하는지 확인하기

1. **웹사이트 접속**
   - Netlify에서 제공받은 주소로 접속
   - 팀 목록 페이지가 나타나는지 확인

2. **기능 테스트**
   - [ ] 팀 카드 클릭해보기
   - [ ] 게시글 작성 버튼 클릭해보기
   - [ ] 모바일에서도 접속해보기

3. **주소 공유**
   - 완성된 웹사이트 주소를 팀원들과 공유
   - 북마크에 저장하여 쉽게 접속

## 🌟 웹사이트 기능

### 📱 메인 페이지
- **팀 목록**: 6개 팀 (Frontend, Backend, Design, DevOps, Product, QA)
- **인기 팀**: 활동이 많은 팀에 '인기' 배지 표시
- **팀 정보**: 각 팀의 멤버 수와 최근 활동 시간

### 📝 팀별 게시판
- **게시글 목록**: 제목, 작성자, 작성시간, 좋아요 수
- **검색 기능**: 제목이나 내용으로 게시글 검색
- **새 글쓰기**: 간편한 글 작성 폼

### ✍️ 게시글 작성
- **제목/내용 입력**: 직관적인 입력 폼
- **작성 팁**: 좋은 글 작성을 위한 가이드
- **실시간 확인**: 입력하면서 바로 확인 가능

### 👀 게시글 보기
- **전체 내용**: 게시글 전문 읽기
- **댓글 기능**: 댓글 작성 및 확인
- **좋아요**: 마음에 드는 글에 좋아요 표시

## 🚨 문제 해결

### ❓ 자주 묻는 질문

**Q: 배포 중에 오류가 발생했어요**
- A: Netlify의 `Deploys` 탭에서 오류 로그 확인
- A: 빌드 명령어가 `npm run build`인지 확인
- A: Node.js 버전이 18 이상인지 확인

**Q: 웹사이트가 안 열려요**
- A: 배포가 완전히 끝났는지 확인 (녹색 체크 표시)
- A: 인터넷 연결 상태 확인
- A: 다른 브라우저에서 시도해보기

**Q: 폰트가 이상하게 나와요**
- A: 정상입니다! 시스템 기본 폰트를 사용하도록 설정되어 있습니다
- A: 컴퓨터마다 조금씩 다르게 보일 수 있습니다

**Q: 모바일에서 화면이 깨져요**
- A: 브라우저 새로고침 (F5) 시도
- A: 캐시 삭제 후 다시 접속

### 🔧 기술적 문제

**Build 실패 시:**
```bash
# 로컬에서 빌드 테스트

```
